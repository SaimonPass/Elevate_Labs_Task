IP for my network interface is 192.168.100.25/24

There are many other Nmap scan options available on Nmap; to know some of them we need to type this command on terminal

nmap -h

Shows all the parameters that can be used for scanning a network.

man nmap

Shows the manual page of Nmap.

In the above screenshot we can see that there are many types of can we can perform on our network.

Below is the command I am going to run in the terminal

sudo nmap -sS -sC -sV 192.168.100.0/24 -T4

-sC Stands for script scanning.

-sV Stand for service version scanning and scans for services running on the port.

-T4 it is an timing template on Nmap optimized for scanning speed.

To run these command we need sudo permission.

The current service running on my local machine is dnsmasq 2.90 on port number 53/tcp that is in (open) state.

Dnsmasq 2.90 is a lightweight, easy-to-configure DNS forwarder and DHCP server, primarily used for small networks. It provides DNS and DHCP services, enabling local machines to be recognized within the network without relying on global DNS servers.

After researching through this Service Version I have found out that this port is vulnerable to Integer Overflow via forward query.
